Closed-Loop Control for Distributed Lithography

Distributed writers diverge without local feedback. The Phoenix engine maintains competing causal hypotheses for process drift, weights them against in-situ sensor data, and derives correction factors within the exposure loop. Without this, multi-writer systems accumulate uncorrelated drift across heads.

Phoenix is a multi-hypothesis state estimator operating inside the exposure loop. It tracks three competing explanations — nominal operation, thermal drift, and source aging — and updates weights from live sensor data. Conventional lithography assumes static process conditions and corrects offline. That assumption breaks when writers operate in parallel.

Measured behavior: without continuous evaluation, writer-to-writer dose variance exceeds tolerance within minutes of thermal transient onset. Early commitment to a single process model amplifies this error at every node. Phoenix holds hypotheses open until convergence, trading compute for robustness.

Consequence: real-time dose compensation across distributed writer arrays, graceful degradation under partial sensor failure, and stable convergence as writer count scales.

State Variables

The Phoenix engine tracks a set of continuous state variables for each hypothesis and for the system as a whole. These variables evolve on every control loop tick in response to sensor data and internal update rules.

Variable Type Range Update Rule Purpose
w_i float 0 to 1 Normalized from likelihood score Hypothesis weight; controls correction factor blending
c_i float 0 to 1 Exponential decay + residual gain Confidence in hypothesis; gates active status
age_i int 0 to T_max age_i = age_i + 1 each tick Counts persistence; triggers retirement when age > T_max
v_i float[n] unbounded Kalman-like prediction update State vector (drift, temp, aging coeff); used in residual calc
eligible_i bool true/false if c_i > τ_merge and age_i > τ_age then true Flags if hypothesis can merge with others
residual_i float 0 to ∞ ||y - h(v_i)|| Error between observation and predicted state
Update Logic

On each control loop tick, Phoenix executes a deterministic sequence of updates. Sensor data enters at the top; correction factors exit for actuation.

1. INPUT: Read photodiode signal y from sensor array
2. PREDICT: For each active hypothesis i, compute predicted state h(v_i) and residual_i = ||y - h(v_i)||
3. WEIGHT: Compute likelihood L_i = exp(-residual_i² / σ²); normalize w_i = L_i / ∑L_j
4. CONFIDENCE: Update c_i = λ·c_i + (1-λ)·w_i; apply exponential decay if w_i < θ_decay
5. MERGE CHECK: For each pair (i, j) where eligible_i ∧ eligible_j, compute distance D_ij; merge if D_ij < τ_merge_dist
6. SPAWN: If all w_i are low (concentration < 0.5) and count < N_max, spawn new hypothesis from sensor residual
7. RETIRE: For each hypothesis, if c_i < τ_retire or age_i > T_max, mark inactive and remove from active set
8. SELECT: Compute system state as v_system = ∑(w_i · v_i) and correction factor Δ from system state
9. OUTPUT: Apply Δ to writer drive signals; store telemetry and log state

Key Thresholds: θ_decay (confidence below which decay accelerates, default 0.3), τ_merge (merge eligibility confidence, default 0.6), τ_retire (retirement confidence floor, default 0.05), T_max (max age in ticks, default 500), N_max (max active hypotheses, default 5), τ_merge_dist (state vector distance, adaptive).

Failure and Recovery Pathologies

The continuous evaluation strategy introduces distinct failure modes. Each has a detection mechanism and a recovery protocol.

Pathology Condition Detection Recovery Severity
Premature Collapse Single hypothesis reaches w=0.95 before age threshold Monitor Herfindahl index; flag if > 0.9 Enforce minimum persistence; re-spawn low-weight hypotheses RED
Merge Pathology Distance metric τ_merge_dist too loose; unrelated hypotheses fuse Track residual before/after merge; reject if Δresidual > threshold Revert merge; increase τ_merge_dist; spawn new hypothesis AMBER
Starvation Too few active hypotheses (<2); system loss of redundancy Monitor active count; alert if < N_min (typically 2) Spawn new hypothesis from random perturbation of best state AMBER
Oscillation Weights cycle without convergence; correction flips sign repeatedly Track correction Δ sign changes over window of 50 ticks; threshold >5 Increase confidence decay rate (lower λ); retire lowest-confidence hypothesis AMBER
Sensor Fault Photodiode saturated or dead; all residuals equal/constant Variance of residual < threshold; or saturation flag from sensor driver Freeze weight updates; broadcast alert; fall back to nominal correction Δ=1.0 GREEN
